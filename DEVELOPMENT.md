## Environment Variables

Copy `.env.example` to `.env` and configure the following:

```dotenv
APP_NAME="Refugee Needs System"
APP_ENV=local
APP_KEY=                        # generated by php artisan key:generate
APP_DEBUG=true
APP_URL=http://localhost:8000

# Database — PostgreSQL
DB_CONNECTION=pgsql
DB_HOST=127.0.0.1
DB_PORT=5432
DB_DATABASE=refugee_needs_system
DB_USERNAME=postgres
DB_PASSWORD=your_password

# Session and cache use the database driver
SESSION_DRIVER=database
CACHE_STORE=database
QUEUE_CONNECTION=database

# SMS gateway (AfricasTalking or similar)
SMS_API_URL=https://api.example.com/sms/send
SMS_API_TOKEN=your_bearer_token

# Set true in development to skip actual SMS sending.
# OTP will be logged and returned in API responses.
SMS_BYPASS=true
```

`config/services.php` exposes the SMS config under the `sms` key:

```php
'sms' => [
    'url'    => env('SMS_API_URL'),
    'token'  => env('SMS_API_TOKEN'),
    'bypass' => env('SMS_BYPASS', false),
],
```

## Authentication Flows

### Refugee (phone + password)

```
POST /login
  body: { phone_no, password }
  ↓
LoginController::login()
  → Hash::check(password, refugee->password)
  → session(['refugee_id', 'refugee_phone', 'refugee_name'])
  → JSON { status: 'success' }
  ↓
JS redirects to /dashboard
```

Session is guarded by `EnsureRefugeeAuthenticated` middleware  
(alias: `refugee.auth` in `bootstrap/app.php`).

### Refugee Registration (phone + OTP + password)

```
Step 1 — POST /register
  body: { name, phone_no, date_of_birth, country_of_origin, host_country, password, password_confirmation }
  ↓
RegistrationController::store()
  → validate (incl. Password::min(8)->letters()->mixedCase()->numbers()->symbols())
  → OtpService::generate() + send()
  → Cache::put("pending_registration_{phone}", validatedData, 10 min)
  → JSON { status: 'otp_sent', dev_otp? }   ← dev_otp only when SMS_BYPASS=true

Step 2 — POST /register/verify-otp
  body: { phone_no, otp }
  ↓
RegistrationController::verifyOtp()
  → OtpService::verify()
  → Refugee::create(cachedData)             ← 'hashed' cast auto-bcrypts password
  → session([refugee_id, …])
  → JSON { status: 'registered' }
```

### Staff (email + password)

```
POST /staff/login
  body: { email, password }
  ↓
AdminAuthController::login()
  → Auth::attempt()
  → session()->regenerate()
  → AuditLogService::log('login', …)
  → redirect /dashboard
```

Laravel's built-in `auth` middleware guards all staff routes.

## Priority Scoring Algorithm

Implemented in `app/Services/PriorityService.php`.

```
priority_score = (urgency_level × 20 + vulnerability_bonus) × category_weight
```

### Vulnerability bonuses

| Condition | Bonus |
|---|---|
| Age < 5 or > 65 | +20 |
| `has_critical_health` | +30 |
| `has_disability` | +15 |
| `is_pregnant` | +20 |
| `family_size` > 5 | +10 |

### Category weights

| Category | Weight |
|---|---|
| healthcare | 1.5 |
| protection | 1.4 |
| food | 1.3 |
| shelter | 1.2 |
| education | 1.0 |

### Maximum score

```
(5×20 + 20+30+15+20+10) × 1.5 = (100 + 95) × 1.5 = 292.50
```

Score thresholds used in the UI:

| Range | Colour | Severity |
|---|---|---|
| ≥ 200 | Red | Critical |
| 100 – 199 | Amber | High |
| < 100 | Emerald | Moderate |

## Services

### `OtpService`

| Method | Description |
|---|---|
| `generate(phone)` | Generates 6-digit OTP, caches `sha256` hash for 10 min. In bypass mode also caches plain value. |
| `send(phone, otp)` | In bypass mode: `Log::info`. Otherwise: HTTP POST to `SMS_API_URL` with bearer token. Throws `RuntimeException` on non-2xx. |
| `verify(phone, otp)` | Compares `hash('sha256', $otp)` against cached hash. Deletes both cache keys on success. |
| `getBypassOtp(phone)` | Returns plain OTP from cache (bypass mode only). |

Cache key pattern: `otp_{phone}` (hash), `otp_plain_{phone}` (bypass plain).

### `PriorityService`

Single public method: `calculate(array $data, Refugee $refugee): float`  
Called by `NeedController::store()` and `NeedController::update()`.

### `AuditLogService`

Single public method: `log(event, auditableType, auditableId, oldValues, newValues, refugeeActorId)`  
Reads `Auth::id()` for `performed_by` and `request()->ip()` for the IP address.

## RBAC

Powered by `spatie/laravel-permission`. Roles are seeded — do not hard-code IDs.

| Role | Description |
|---|---|
| `admin` | Full access including audit logs and user management |
| `aid_worker` | Can record and manage needs; read-only reports |

Middleware aliases registered in `bootstrap/app.php`:

```php
'refugee.auth' => EnsureRefugeeAuthenticated::class,
'role'         => \Spatie\Permission\Middleware\RoleMiddleware::class,
'permission'   => \Spatie\Permission\Middleware\PermissionMiddleware::class,
```

## Development Workflow

### Start the dev environment (all-in-one)

```bash
composer dev
```

This runs three processes concurrently via `concurrently`:
- `php artisan serve` — PHP dev server on port 8000
- `npm run dev` — Vite dev server with HMR
- `php artisan queue:listen --tries=1` — processes queued jobs

### Run individually

```bash
php artisan serve
npm run dev
```

### Tailwind CSS

Uses Tailwind v4 with the official `@tailwindcss/vite` plugin — **not** the PostCSS plugin.

```js
// vite.config.js
import tailwindcss from '@tailwindcss/vite';
plugins: [tailwindcss(), laravel({...})]
```

```css
/* resources/css/app.css */
@import "tailwindcss";
@source "../**/*.blade.php";
```

Do not add `tailwindcss: {}` to `postcss.config.js` — it conflicts with the Vite plugin.

### Artisan commands used in this project

```bash
php artisan migrate              # run all pending migrations
php artisan migrate:rollback     # undo last batch
php artisan db:seed              # runs CountrySeeder + RolesAndPermissionsSeeder
php artisan db:seed --class=RefugeeSeeder
php artisan route:list           # inspect all registered routes
php artisan cache:clear
php artisan config:clear
```
